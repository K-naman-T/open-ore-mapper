# Security

## Sensitive Data

Do not upload private exploration data, coordinates, survey cubes, credentials, or proprietary spectra to public issues, discussions, or pull requests.

## Reporting Vulnerabilities

After the GitHub repository is created, report vulnerabilities through GitHub private vulnerability reporting. Until then, contact the maintainers privately.

## Supported Versions

The project is pre-1.0. Security fixes are applied to the latest commit on the default branch.
